
## Data Flow

0-1 '공고유형': notices 테이블 category
  1) '공사점검'
  2) '성능평가
  3) '기타'
  4) '무관': 회사업무와 무관, default값

0-2 '입찰단계': notices 테이블 status
  1) '제외': 입찰 진행에서 제외 or 응찰전 포기, default값
  2) '진행': 입찰 진행
  3) '응찰': 응찰
  4) '포기': 응찰 후 포기
  5) '낙찰'
  6) '패찰'

1. 입찰 목록 스크랩 설정에 따라 매일 2회 입찰 공고 게시판 목록 스크랩 및 데이터베이스(notices)에 저장
  - 게시판 목록 중 회사 업무 관련 공고인 경우, 카테고리 설정(settings_category)에 따라 공고유형(category: '공사점검', '성능평가', '기타') 지정, 입찰단계(status)는 default값인 '제외'
  - 회사 업무와 관련 없는 경우, 공고유형은 '무관'으로 지정, 입찰단계(status)는 의미가 없으나 default값인 '제외'로 저장

2. 1단계의 결과에 따라, '공고 목록'의 아래와 같은 테이블에 나타남
  1) '공사점검'
  2) '성능평가
  3) '기타'
  4) '무관'

3. 공고 목록 페이지에서
  1) 공고 유형 변경:
   - 우측 상단의 '수정' 아이콘 버튼을 누르면, '유형 변경' 모달이 나타남
   - 변경 후 저장을 클릭하면, 데이터베이스의 notices 테이블의 category값 update 후, UI에도 즉시 적용
  2) 입찰 단계 변경
   - 우측 상단의 '즐겨찾기에 추가' 버튼을 누르면, '입찰을 진행합니다' 프롬프트 창이 나타남
   - '예'을 클릭하면, 
     - 데이터베이스의 notices 테이블의 status값을 '진행'으로 update
     (- 데이터베이스의 bids 테이블에 관련 정보가 insert됨
     - 공고 상세페이지 스크랩(페이지 내용 + 첨부 파일 다운로드)
     - UI에도 즉시 적용(현재 공고 목록에서는 제거되고, '입찰 관리'의 '진행' 목록 페이지에는 추가됨)

4. '입찰 관리'(status: 진행, 응찰, 포기, 낙찰, 패찰)
  - 진행(입찰 준비중인 공고 목록)
    - 목록
      - '수정' 아이콘 클릭: '입찰 단계 변경'(응찰, 제외) 모달
    - 진행 페이지: 해당 입찰 공고 클릭시
      - 상세 페이지 내용
      - [예정]공고 문서 보기
      - [예정]응찰용 문서 작성
  - 응찰 목록
    - 목록
      - '수정' 아이콘 클릭: '입찰 단계 변경'(낙찰, 패찰, 진행, 포기) 모달
    - 진행 페이지: 해당 입찰 공고 클릭시
      - 상세 페이지 내용
      - [예정]공고 문서 보기
      - [예정]입찰 결과 확인 페이지
  - 낙찰 목록
    - 목록
      - '수정' 아이콘 클릭: '입찰 단계 변경'(패찰, 진행, 응찰, 제외, 포기) 모달
    - 진행 페이지: 해당 입찰 공고 클릭시
      - [예정]공고 문서 보기
      - [예정]프로젝트 페이지
  - 패찰 목록
    - 목록
      - '수정' 아이콘 클릭: '입찰 단계 변경'(낙찰, 진행, 응찰, 제외, 포기) 모달
