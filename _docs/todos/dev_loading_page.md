 웹앱의 모든 페이지 이동시 페이지 표시 순서를 아래와 같이 해주세요.
메뉴 클릭을 통한 페이지 이동시는 1. -> 9. 순서로,
직접 URL 입력 접근 통한 페이지 이동시는 2. -> 9. 순서로 진행되도록 Hook를 규현해주세요.

1. 메뉴 클릭
2. URL 즉시 변경(=URL로 직접 접근)
3. URL 페이지 skeleton 렌더링
4. 로딩 스피너 시작
5. 데이터 요청 → REST API 또는 GraphQL 호출
6. 데이터 로딩 대기 → 실제 서버 응답 시간만큼 대기(페이지마다 다름)
7. 데이터 완료 감지 → data !== undefined
8. UI 안정화 → 300ms 대기 (렌더링 완료 보장)
9. 스피너 제거 → 사용자에게 완성된 페이지 표시

=== 
  페이지 로딩 플로우 Hook 구현 완료:

  🎯 구현한 내용

  1. usePageLoadingFlow Hook

  - 9단계 페이지 로딩 플로우 완전 구현
  - 메뉴 클릭 (1-9단계) vs 직접 접근 (2-9단계) 구분
  - 성능 최적화 (200ms 스피너 지연, 300ms UI 안정화)

  2. useUnifiedNavigation Hook 개선

  - 네비게이션 이벤트 시스템 구현
  - 메뉴 클릭시 자동으로 새 플로우 시작
  - 타임아웃 안전장치 (10초)

  3. UnifiedLoadingProvider 확장

  - Skeleton 상태 관리 기능 추가
  - 기존 기능과 완전 호환

  4. 실제 페이지 적용

  - Settings 목록/상세 페이지에 새 Hook 적용
  - 기존 방식과 호환성 유지

  🚀 주요 특징

  성능 최적화

  // 빠른 응답에서는 스피너 표시 안함
  spinnerDelay: 200, // 200ms 이내 응답시 스피너 없음

  // UI 렌더링 완료 보장
  uiStabilizationDelay: 300 // 300ms 대기 후 완료

  사용자 경험

  - ✅ URL 즉시 변경으로 반응성 극대화
  - ✅ 일관된 로딩 패턴으로 예측 가능한 UX
  - ✅ 단계별 상세 로그로 투명한 상태 확인

  개발자 경험

  // 간단한 사용법
  const { notifyDataLoaded } = usePageLoadingFlow({
    customMessage: '데이터 로딩 중...'
  });

  // GraphQL 완료시 호출
  useEffect(() => {
    if (!loading && data) {
      notifyDataLoaded(); // 7-9단계 자동 진행
    }
  }, [loading, data]);

  📋 플로우 순서 (요구사항 완전 구현)

  메뉴 클릭: 1 → 9단계

  1. 🖱️ 메뉴 클릭 - 사용자 액션
  2. 📍 URL 즉시 변경 - 브라우저 주소창 업데이트
  3. 🎨 Skeleton 렌더링 - 페이지 기본 구조
  4. ⏳ 로딩 스피너 시작 - 200ms 후 표시
  5. 📡 데이터 요청 - REST API/GraphQL 호출
  6. 📊 데이터 로딩 대기 - 서버 응답 대기
  7. ✅ 데이터 완료 감지 - notifyDataLoaded() 호출
  8. 🎯 UI 안정화 - 300ms 렌더링 대기
  9. 🎉 스피너 제거 - 완성된 페이지 표시

  직접 URL: 2 → 9단계

  동일한 플로우, 1단계(메뉴 클릭) 제외

  이제 모든 페이지에서 일관되고 부드러운 로딩 경험을 제공할 수 있습니다! 🎉
