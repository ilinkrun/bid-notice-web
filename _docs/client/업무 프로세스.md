<입찰업무 프로세스>

1. 입찰 홈페이지에서 '스크랩에러' 와 날자별 공고통계를 보고 현황을 파악한다.
  '스크랩에러' 에 등록된 기관은 매뉴얼로 공고를 확인하여야 되는 것으로 인지, 수동확인 한다.

2. '공고목록'에서 Assign(진행, 무관, 포기)이 되지 않은
   신규 등록건을 클릭하여 공고 사이트에서 확인한다.

3. 공고 내용 확인후 '공고목록'에서
   1) Assign(진행, 무관, 포기) 한다.(사용자)

     '진행': 입찰 진행할 건
             '입찰관리'의 '입찰데이터' 목록에 추가함(프로그램)
     '포기': 공고목록에는 있고 입찰요건 해당하나, 입찰을 하지 않는 것으로 결정한 건
             '입찰관리'의 '포기'건 목록에 추가함(프로그램)
     '무관': 공고목록에는 있느나, 입찰요건에 해당하지 않는 건
             '입찰관리'의 '무관'건 목록에 추가함(프로그램)

      ; 위 지정은 '공고목록'에 '구분' 칸에서 보여 짐.(프로그램)

    2) '진행'으로 어싸인 하면, 폴더를 작성 수 있게
        가). Root 폴더를 선택하게 한다(프로그램); 파일 창으로 띄워서
             - Default 폴더: Z:\24_공사점검\3 . 용역 입찰공고
             - 폴더 없으면 상위 폴더..
        나) 하위 기관 폴더명은
             - 기본 Default로 '[회사명] 번호. 수행기관' 형식으로 한다.
             (예;[일맥] 02. 수원시, [링크] 01.인천해양경찰서)
             <= 기존에 폴더가 있으면 새로 작성하고
             <= 숫자까지 정확히 일치하는 것으로 찾는 것이 아니고,
                   회사와 기관명 포함단어로 기존 폴더가 있는지 확인함
             <=없으면, 해당 회사별  '최대 일련번호+1'로 번호 매겨
                  폴더명을 입력창 텍스트창에 가지고 온다.
            - 기관 폴더명은 Default값에서 사용자가 수정할 수 있게 한다.
        다) 용역명 폴더명(공사명)(기관 폴더명 하위)을 작성한다.
             - 기본 Default로 '번호. 용역명(혹은 공사명)' 형식으로 한다.
                  (예;2. 서울시 양천구 목동 231-31 근생 및 다중주택 신축공사)
                <= 기존에 다른 폴더가 있으면 '최종 일련번호+1', 없으면 '1.' 용역명으로 작명함.
                <=공사명을 앱이 Default값으로 텍스트 입력칸에 입력한다.
                <=사용자가 최종 수정한 값으로 '폴더생성' 한다.
         라) 서브폴더 작성
             - '01 입찰자료' 생성하고
             - 그 하위에 '01 접수자료', '02 작성서류',  '03 제출서류'


4. 공고내용 중 첨부파일을 다운.
    - 작성된 폴더 '01 입찰자료' 에 첨부 문서를 다운

5. 첨부 문서 분석하여 '입찰데이터'에
   - '접수방법', '개시', '마감일', '기초금액', '낙찰 하한율'  등 입찰조건을 입력
   -'단계'는 데이타가 넘어오면 '진행' 으로 선택됨(프로그램)

6. 입찰 제출서류 를 파악하여 입찰서 작성

7. 투찰가 정하여  입찰데이타에 '투찰가'를 기록하고 첨부 문서를 송부하면
   '단계'를 '완료' 변경 선택

8. 중간에 입찰을 포기하면 '단계'를 '포기'로 변경 선택입력하여야 함.

9 투찰 완료한 입찰 건이 '낙찰'이면 상태 변경입력, 낙찰이 안되면 변경 기록 없음.
   개찰 날짜가 지나 '완료' 상태인 것은 패찰이라는 의미임.

-----------------------------------------------------------------------------------------------------

10. 낙찰이 되면 '프로젝트 현황' 목록에 추가되고,(추후 프로 그램)

11.  기작성된 폴더를 각 프로젝트 구분별 '진행'  폴더에 이관하며, 용역 종류별 표준폴더를 기준으로
      폴더를 추가 생성한다.(추후 프로 그램)
